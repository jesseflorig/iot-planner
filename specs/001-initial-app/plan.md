# Implementation Plan: IoT Planner — Initial Application

**Branch**: `001-initial-app` | **Date**: 2026-05-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-initial-app/spec.md`

## Summary

A fully client-side web app for planning IoT device builds on Feather-form-factor boards.
The primary interface is an SVG breadboard view showing the active board and any added modules,
with pin status (available / in-use / error) rendered directly in the SVG. Built with React 18 +
TypeScript + Tailwind CSS + Zustand, mirroring the bracket-generator architecture.

## Technical Context

**Language/Version**: TypeScript 5.7 + React 18.3
**Primary Dependencies**: Vite 5.4, Zustand 5.0, Zod 3.23, Tailwind CSS 3.4, Vitest 2.1
**Storage**: Browser localStorage (key: `iot-planner-package`)
**Testing**: Vitest (globals mode, node environment — same as bracket-generator)
**Target Platform**: Modern web browser; fully offline after initial load; served as static files
**Project Type**: Single-page web application
**Performance Goals**: SVG re-render < 100ms on any package change; tooltip latency < 50ms on hover
**Constraints**: No server, no user accounts, no external API calls; pure client-side
**Scale/Scope**: 2 boards, 1 module (initial release); data model extensible for future additions
